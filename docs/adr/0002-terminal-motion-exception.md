# Terminal layout avoids theatrical terminal motion

Terminal is a terminal-inspired Layout Direction, but its content should feel like readable output that is already available rather than a simulation that makes visitors wait. It reuses the shared layout switcher, theme toggle, content data, normal scrolling, and may use restrained reveal-on-scroll, but it deliberately avoids count-up animation and typewriter delays because those effects fight the scrollback transcript model and add accessibility and scanning costs.
